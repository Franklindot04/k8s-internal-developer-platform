# Code of Conduct

## Our Commitment

We are committed to maintaining a respectful, professional, and collaborative environment for everyone contributing to this project.

## Expected Behavior

Contributors are expected to:

* Communicate respectfully
* Give constructive feedback
* Focus criticism on the work, not the person
* Be patient with questions and different experience levels
* Collaborate in good faith
* Respect project decisions after discussion

## Unacceptable Behavior

Unacceptable behavior includes:

* Harassment, insults, or personal attacks
* Discriminatory language or behavior
* Threats or intimidation
* Sharing private information without permission
* Repeatedly disrupting discussions or reviews
* Dismissing feedback in an unprofessional way

## Project Discussions

Disagreements should be handled through respectful discussion in issues, pull requests, or team communication channels.

Technical decisions should be based on project goals, maintainability, security, platform reliability, and long-term operational value.

## Reporting

If you experience or observe unacceptable behavior, report it to the project lead or one of the project maintainers.

Current project team:

* Ajero Franklin
* Mordecai Nathan
* Escar Maris

Reports should be handled privately, respectfully, and fairly.

## Enforcement

Maintainers may take appropriate action, including:

* Requesting changes to communication
* Removing inappropriate comments
* Closing issues or pull requests
* Restricting contribution access if necessary

## Scope

This Code of Conduct applies to all project spaces, including:

* Repository issues
* Pull requests
* Code reviews
* Documentation discussions
* Team communication related to this project
